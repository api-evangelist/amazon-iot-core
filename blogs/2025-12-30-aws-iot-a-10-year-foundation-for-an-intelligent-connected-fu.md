<p>December 2025 marks ten years since AWS for Internet of Things (<a href="https://aws.amazon.com/iot-core/?nc=sn&amp;loc=1">AWS IoT</a>) set out to connect the physical world to the digital world. Today, hundreds of millions of devices connect to AWS IoT services daily, enabling organizations to extract real-time insights, automate processes, and unlock entirely new business models across virtually every industry. This includes powering smart consumer and commercial products, connected vehicles, industrial automation, precision agriculture, and intelligent healthcare systems worldwide. This blog is a reflection of our journey with customers over the past ten years and where we see the future of IoT heading.</p> 
<h2>AWS IoT: Portfolio of production-ready IoT services that power end-to-end IoT solutions across every industry</h2> 
<p>We started in 2015 with a <a href="https://aws.amazon.com/iot-core/">core set of IoT functions</a> built on a secure and scalable message broker to help customers build production IoT solutions. Since launch, customer feedback has been our driving source of innovation. First, we added <a href="https://aws.amazon.com/blogs/iot/just-in-time-registration-of-device-certificates-on-aws-iot/">Device Shadows</a>, giving every device a persistent “digital twin” in the cloud so applications could read and change device state, even when the device was offline. In 2016, we introduced <a href="https://aws.amazon.com/blogs/aws/aws-greengrass-ubiquitous-real-world-computing/">AWS IoT Greengrass</a>, bringing AWS IoT’s programming model down to gateways and edge devices so customers could run code, process data, and act locally when connectivity is limited. In 2017, we launched <a href="https://aws.amazon.com/blogs/aws/aws-iot-device-management/">AWS IoT Device Management</a> to make it easier to onboard, organize, monitor, and update fleets of devices through their full lifecycle, and took stewardship of <a href="https://aws.amazon.com/blogs/aws/announcing-freertos-kernel-version-10/">FreeRTOS</a>, a real-time operating system for resource-constrained devices. That same year, we introduced <a href="https://aws.amazon.com/blogs/aws/amazon-kinesis-video-streams-serverless-video-ingestion-and-storage-for-vision-enabled-apps/">Amazon Kinesis Video Streams</a> to enable customers to securely stream video from connected devices to AWS for analytics, machine learning, and other processing. In 2019, we added <a href="https://aws.amazon.com/blogs/iot/new-fleet-indexing-enables-you-to-index-search-and-aggregate-your-device-data/">Fleet Indexing</a> in IoT Device Management, enabling customers to search and aggregate across registry data, shadows, and connectivity state so they can quickly answer questions like “which devices are online and on which firmware?” and take action at fleet scale. We also launched <a href="https://aws.amazon.com/blogs/aws/aws-iot-sitewise-collect-organize-and-analyze-equipment-data/">AWS IoT SiteWise</a>, making it easier to collect, organize, and analyze data from industrial equipment at scale.</p> 
<p>In 2020, we launched <a href="https://aws.amazon.com/blogs/iot/how-to-automate-onboarding-of-iot-devices-to-aws-iot-core-at-scale-with-fleet-provisioning/">Fleet Provisioning</a> in AWS IoT Core, so manufacturers and operators could securely onboard large volumes of devices automatically using templates instead of manual, one-by-one registration. In 2021, we introduced <a href="https://aws.amazon.com/blogs/aws/new-aws-iot-twinmaker-build-digital-twins-of-real-world-systems/">AWS IoT TwinMaker</a>, enabling customers to create digital twins of real-world systems to optimize operations and improve performance. The following year, in 2022, we launched <a href="https://aws.amazon.com/blogs/iot/announcing-aws-iot-expresslink/">AWS IoT ExpressLink</a>, providing a range of hardware modules that simplify and accelerate secure cloud connectivity for device makers. We also announced <a href="https://aws.amazon.com/blogs/iot/collecting-vehicle-data-more-efficiently-with-aws-iot-fleetwise-2/">AWS IoT FleetWise</a>, making it easier and more cost-effective for automotive companies to collect, transform, and transfer vehicle data to the cloud in near-real time. In 2024, AWS IoT introduced <a href="https://aws.amazon.com/blogs/iot/simplify-iot-application-development-with-message-enrichment-for-aws-iot-core/">message enrichment</a> to help customers dynamically interact with messages without firmware changes, developed a new <a href="https://aws.amazon.com/blogs/iot/announcing-webrtc-sdk-for-amazon-kinesis-video-streams/">WebRTC SDK for Kinesis Video Streams</a> to speed up camera integrations, and added <a href="https://aws.amazon.com/blogs/iot/mqtt5-support-now-available-in-aws-iot-core/">MQTT v5 support</a> to AWS IoT Core, bringing enhanced features like user properties, message expiry, and improved error reporting. Finally, when customers asked for more flexibility in runtime support and the ability to run on a wider variety of devices, this year AWS IoT responded by releasing the <a href="https://aws.amazon.com/blogs/iot/announcing-aws-iot-greengrass-nucleus-lite/">nucleus lite runtime for AWS IoT Greengrass</a>. This new runtime operates on as little as 5MB of memory, opening up cloud connectivity to an entirely new class of devices. Most recently, in March 2025, we launched <a href="https://aws.amazon.com/blogs/iot/managed-integrations-for-aws-iot-device-management-is-now-generally-available/">managed integrations for AWS IoT Device Management</a>, enabling developers to streamline cloud onboarding of IoT devices and unify device control across multiple brands and protocols with built-in support for ZigBee, Z-Wave, and Wi-Fi specifications, plus 80+ device data model templates and a comprehensive catalog of cloud-to-cloud connectors.</p> 
<p>It’s been inspiring to see the variety of solutions (across virtually every industry domain) that our customers have built using AWS IoT services over the past ten years.</p>

<h2>IoT solutions built to integrate with AWS services and partners for end-to-end solutions</h2> 
<p>Over the years, we’ve adapted and evolved IoT services with deep AWS service integrations that enable our customers to more easily and quickly build end-to-end solutions. For example, IoT workloads can seamlessly integrate with <a href="https://aws.amazon.com/eventbridge/">AWS EventBridge</a>, which provides robust event routing and filtering capabilities with seamless AWS service integration, enabling customers to build sophisticated event-driven architectures from IoT device data. For analytics workloads, IoT data streams can leverage <a href="https://aws.amazon.com/glue/">AWS Glue</a> for ETL processing and <a href="https://aws.amazon.com/athena/">Amazon Athena</a> for interactive queries, creating powerful data lakes on <a href="https://aws.amazon.com/s3/">Amazon Simple Storage Service (Amazon S3)</a> and implementing comprehensive analytics pipelines. These solutions integrate seamlessly with visualization tools like <a href="https://aws.amazon.com/quicksuite">Amazon Quick Suite</a> for actionable AI-powered insights and workflows from IoT telemetry and sensor data. Across AI/ML workstreams, customers can train models using <a href="https://aws.amazon.com/sagemaker/">Amazon SageMaker</a> with IoT datasets, fine-tune them with <a href="https://aws.amazon.com/bedrock/">Amazon Bedrock</a> for generative AI applications, and deploy them at scale using SageMaker endpoints or directly to edge devices through AWS IoT Greengrass—all while maintaining seamless IoT workload integration.</p>
